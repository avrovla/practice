# Требования к системе

## Функциональные требования

| ID | Требование |
|----|------------|
| ФТ1 | Приём текстового запроса на русском/английском языке |
| ФТ2 | Извлечение из запроса ключевых параметров: химический состав / класс материалов |
| ФТ3 | Вызов REST API M3GNet (FastAPI) на выделенном сервере |
| ФТ4 | Передача структур и базовых параметров дообучения |
| ФТ5 | Получение статуса и результата дообучения |
| ФТ6 | Сохранение дообученной модели с именем, переданным пользователем при запуске дообучения |
| ФТ7 | Определение энергии формирования материала с использованием указанной дообученной модели (имя модели — один из входных параметров) |
| ФТ8 | Демонстрация на одном конкретном классе материалов |

## Нефункциональные требования

| ID | Требование |
|----|------------|
| НФТ1 | Модульность (агент, API M3GNet, Ollama — отдельные компоненты) |
| НФТ2 | Возможность запуска агента на нескольких ПК |
| НФТ3 | Асинхронность (отсутствие блокировки пользователя при длительных операциях) |
| НФТ4 | Агент работает на CPU (8–16 GB RAM, GPU не обязателен) |
| НФТ5 | Обработка ошибок (недоступность бэкенда, LLM, отсутствие указанной модели) |
| НФТ6 | Работа в изолированной сети (без выхода в интернет) |
| НФТ7 | CLI-интерфейс |
| НФТ8 | Логирование всех запросов и результатов |
