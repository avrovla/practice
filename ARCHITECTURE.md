# Архитектура системы

## Обзор архитектуры

Система представляет собой модульный CLI-агент для работы с кристаллическими структурами, использующий LLM (Ollama) и M3GNet API для дообучения моделей и предсказания энергии формирования материалов.

```mermaid
graph TB
    User[Пользователь] -->|CLI запрос| Agent[Агент]
    Agent -->|HTTP запрос| M3GNet[M3GNet API]
    Agent -->|HTTP запрос| Ollama[Ollama LLM]
    Agent -->|Запись| Logger[Логгер]
    
    subgraph "Сервер M3GNet"
        M3GNet
    end
    
    subgraph "Сервер Ollama"
        Ollama
    end
    
    subgraph "Локальная машина (клиент)"
        Agent
        Logger
    end
```

## Компоненты системы

### 1. Агент (Agent)

Центральный компонент системы, обрабатывающий пользовательские запросы и координирующий взаимодействие с другими сервисами.

```mermaid
graph LR
    CLI[CLI интерфейс] --> Parser[Парсер запросов]
    Parser -->|Извлеченные параметры| Extractor[Экстрактор параметров]
    Extractor --> Router[Маршрутизатор]
    Router -->|Дообучение| FineTune[Модуль дообучения]
    Router -->|Предсказание| Predict[Модуль предсказания]
    FineTune --> M3GNetClient[M3GNet API клиент]
    Predict --> M3GNetClient
    M3GNetClient --> Response[Формирование ответа]
```

**Ответственность:**
- Приём и парсинг CLI-запросов (ФТ1)
- Извлечение ключевых параметров: химический состав, класс материалов (ФТ2)
- Маршрутизация запросов к соответствующим модулям
- Взаимодействие с M3GNet API (ФТ3, ФТ4)
- Получение статуса и результатов дообучения (ФТ5)
- Сохранение дообученной модели (ФТ6)
- Предсказание энергии формирования (ФТ7)
- Асинхронная обработка запросов (НФТ3)
- Обработка ошибок (НФТ5)
- Логирование всех операций (НФТ8)

### 2. M3GNet API (FastAPI)

REST API для работы с моделью M3GNet, развёрнутый на выделенном сервере.

```mermaid
graph LR
    API[FastAPI Endpoint] --> FT[Дообучение модели]
    API --> PE[Предсказание энергии]
    API --> MS[Управление моделями]
    FT --> ModelStore[(Хранилище моделей)]
    PE --> ModelStore
    MS --> ModelStore
```

**Ответственность:**
- Приём запросов на дообучение модели (ФТ3, ФТ4)
- Выполнение дообучения с переданными параметрами
- Предсказание энергии формирования материала (ФТ7)
- Сохранение и управление дообученными моделями (ФТ6)
- Возврат статуса операций (ФТ5)

### 3. Ollama LLM

Локальная внутри сети языковая модель для обработки естественного языка.

```mermaid
graph LR
    Query[Текстовый запрос] --> OllamaAPI[Ollama API]
    OllamaAPI --> NLP[Обработка языка]
    NLP --> Params[Извлеченные параметры]
```

**Ответственность:**
- Обработка запросов на русском/английском языке (ФТ1)
- Помощь в извлечении ключевых параметров из запроса (ФТ2)

### 4. Логгер (Logger)

Компонент для логирования всех операций системы.

```mermaid
graph LR
    LogEntry[Запись в лог] --> Formatter[Форматирование]
    Formatter --> File[Файл логов]
    Formatter --> Console[Консоль]
```

**Ответственность:**
- Логирование всех запросов (НФТ8)
- Логирование результатов операций (НФТ8)
- Логирование ошибок (НФТ5)

## Взаимодействие компонентов

### Сценарий 1: Дообучение модели

```mermaid
sequenceDiagram
    participant U as Пользователь
    participant A as Агент
    participant O as Ollama
    participant M as M3GNet API
    participant L as Логгер
    participant S as Хранилище моделей

    U->>A: CLI запрос на дообучение
    A->>L: Логирование запроса
    A->>O: Обработка естественного языка
    O-->>A: Извлеченные параметры
    A->>A: Извлечение хим. состава и параметров (ФТ2)
    A->>M: POST /fine-tune (ФТ3, ФТ4)
    M-->>A: Task ID
    A->>U: Статус: дообучение запущено
    M->>M: Процесс дообучения
    M->>S: Сохранение модели (ФТ6)
    M-->>A: Дообучение завершено
    A->>U: Уведомление о завершении (ФТ5)
    A->>L: Логирование результата
```

### Сценарий 2: Предсказание энергии формирования

```mermaid
sequenceDiagram
    participant U as Пользователь
    participant A as Агент
    participant O as Ollama
    participant M as M3GNet API
    participant L as Логгер

    U->>A: CLI запрос на предсказание
    A->>L: Логирование запроса
    A->>O: Обработка естественного языка
    O-->>A: Извлеченные параметры
    A->>A: Извлечение структуры и имени модели (ФТ2, ФТ7)
    A->>M: POST /predict (ФТ7)
    M-->>A: Энергия формирования
    A->>U: Результат предсказания
    A->>L: Логирование результата
```

## Развёртывание

```mermaid
graph TB
    subgraph "Клиентские ПК (несколько)"
        Agent1[Агент ПК 1]
        Agent2[Агент ПК 2]
        AgentN[Агент ПК N]
    end
    
    subgraph "Сервер Ollama"
        Ollama[Ollama]
    end
    
    subgraph "Выделенный сервер M3GNet"
        M3GNet[M3GNet API]
        Models[(Хранилище моделей)]
    end
    
    Agent1 --> Ollama
    Agent2 --> Ollama
    AgentN --> Ollama
    
    Agent1 --> M3GNet
    Agent2 --> M3GNet
    AgentN --> M3GNet
    
    M3GNet --> Models
```

**Особенности развёртывания:**
- Агент может запускаться на нескольких ПК (НФТ2)
- M3GNet API работает на выделенном сервере
- Ollama работает на выделенном сервере
- Все компоненты работают в изолированной сети (НФТ6)

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| Агент | Python (CPU, 8-16 GB RAM) |
| M3GNet API | FastAPI |
| LLM | Ollama |
| CLI | argparse / click |
| Логирование | logging / structlog |
| Асинхронность | asyncio |
